# UI/UX Rebuild Plan (2026-02-18)

## ゴール
- 「案件1件を開けば、業務完了まで迷わない」状態を作る。
- 対象フロー: 見積作成 → 請求発行 → 入金消込 → 支払登録/消込 → PDF発行 → 連絡

## 0. 今回の即時修正（完了）
- PDF出力の読込不能を修正（有効なPDF生成 + フォールバック）
- 案件作成失敗時の安定性/文言改善
- KPIを見積・請求・入金・原価・粗利で再定義
- 次アクションバナーを導入

## 1. 48時間で実装するUI/UX
1. 案件作成フォーム
- 必須項目を明示（物件名、顧客）
- 入力エラーを項目直下に表示
- 保存後の遷移/成功トーストを明示

2. 案件コックピット
- 上部: KPI固定（見積/請求/入金/原価/粗利/利益率）
- 中央左: 見積明細
- 中央右: 請求/入金
- 下段右: 支払/消込
- 下段左: 帳票・メール

3. 状態可視化
- ステップ進捗チップ（未着手/進行中/完了）
- 「次にやること」を自動表示

## 2. A4横帳票方針
- 出力デフォルトは A4 Landscape
- 色設定は後段でテーマ化（現段階は機能優先）
- 失敗時は「読める最小帳票」を必ず返す

## 3. 全員共有オペレーション
- ブランチ: `main` 直push（当面）
- 必須: 実装コミット + デプロイログ + 実行ログ
- レビュー用: URLを常に最新版へ更新

## 4. 受入条件
- 案件作成: 10回連続で失敗ゼロ
- 見積PDF: Chrome/Edgeで読込エラー0
- KPI: 明細追加直後に見積合計が反映
- 業務導線: 新人が5分以内に請求発行まで到達
