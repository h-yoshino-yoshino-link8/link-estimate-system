# LinK Estimate System 再設計ブループリント (2026-02-17)

## 1. 目的
- 案件単位で見積・請求・入金・支払・帳票を一気通貫に管理する
- 経営者がトップ画面で資金状態と成長を即把握できる状態にする
- 会計担当が未入金・期限超過・消込作業を横断で処理できる状態にする

## 2. ユーザー別の主要動線

### 経営者
1. `/` を開く
2. KPI（今月売上 / YTD / 前年比 / 売掛残 / 買掛残 / 稼働案件）を確認
3. 期限超過アラートを確認し、必要な案件へ遷移

### 見積担当
1. `/projects` で案件作成または既存案件選択
2. `/projects/{projectId}` で明細追加・見積調整
3. 見積書PDFを出力、案件進行を更新

### 会計/事務
1. `/finance` で未入金・未払・期限超過を横断確認
2. 請求消込/支払消込を更新
3. 領収書出力、督促メール文面作成

## 3. 画面構成
- `/` 経営ダッシュボード
- `/projects` 案件一覧 + 新規案件作成
- `/projects/{projectId}` 案件ワークスペース（見積/請求/支払/帳票）
- `/finance` 会計センター（横断消込・期限管理）

## 4. 情報の親子関係
- 親: Project（案件）
- 子: ProjectItem（見積明細）, Invoice（請求）, Payment（支払）
- 帳票: Estimate PDF / Receipt PDF は Project または Invoice に紐づける

## 5. 現時点での仕様ギャップ
- 本格メール送信API（現状は mailto 文面生成）
- 請求の正式な支払期限カラム（現状は請求日+30日を運用期限として算出）
- 請求書PDF API（現状は見積書PDFと領収書PDF）
- 権限管理（経営/見積/会計の閲覧・更新分離）

## 6. 次フェーズ優先順位
1. Invoice に `due_at` を追加し、期限管理を明示化
2. 請求書PDF API と画面実装
3. メール送信API（送信履歴保存）
4. KPIを月次締めベースに拡張（受注率、平均回収日数、粗利率推移）
